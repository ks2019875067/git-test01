깃 외부에서 돌리는 방법- 원격 저장소 사용
1. git init --bare [name] (현재 디랙토리는 보낼 디랙토리 상위에 있어야 함) : bare형태의 [name]대랙토리를 만듦 bare는 접근용이 아닌 저장용 디랙토리임
2. git remote add [val] [bare-dir] : 현재 디랙토리가 [bare-dir]를 원격저장소로 만들고 경로명을 [val]로 정의함
3. git remote -v : 현재 디랙토리에 원격 연결된 저장소의 경로명,별명을 보여줌
4. git remote remove [val] : 현 디랙토리에 연결된 [val]원격 저장소를 제거함
5. git push : 현 디랙토리에 연결된 원격 저장소로 데이터 전송(해당 설명을 보고 판단 가능)
6. git pull : 현 디랙토리에 연결된 원격 조장소에서 데이터 들고옴(위와 동일판단가능)
외부에서 돌리는 방법-깃허브(원격 저장소와 룰은 큰 차이없음 (1제외) 로그인을 요구할수있음)
1. 위의2~6의 사용그대로 가능
